# Rencana: Menambahkan Entry Checklist di Playbook

## Latar Belakang
User ingin menambahkan checklist item agar user bisa menentukan trading plan sebelum melakukan entry. Ini akan membantu trader untuk disiplin dan terstruktur dalam mengambil keputusan trading.

## Analisis Kebutuhan
- Model Playbook (`Strategy`) sudah memiliki field `entryChecklist` berupa array string
- Saat ini belum ada UI untuk membuat/edit playbook strategy
- Checklist ini akan digunakan sebagai panduan sebelum melakukan entry trading

## Tujuan
1. Membuat UI untuk membuat/edit playbook strategy
2. Menambahkan form input untuk entry checklist (list item yang bisa ditambahkan/dihapus)
3. Menyimpan checklist ke dalam model playbook
4. Menampilkan checklist saat user akan melakukan entry (mungkin di halaman input trade atau sebagai modal konfirmasi)

## Langkah-langkah Implementasi

### 1. Buat Halaman Playbook Form
- Buat file `frontend/src/app/playbook-form/page.tsx` untuk halaman form playbook
- Gunakan layout yang konsisten dengan dashboard
- Implementasi form dengan field:
  - Nama playbook
  - Deskripsi
  - Metodologi (dropdown)
  - Kondisi pasar
  - Timeframe
  - Pasar (array)
  - Aturan (array)
  - **Entry Checklist** (array, dengan kemampuan menambahkan/hapus item)
  - Tags (array)
  - Dll (sesuai model)

### 2. Implementasi Entry Checklist Input
- Gunakan komponen input yang memungkinkan menambahkan item checklist
- Setiap item bisa berupa text biasa
- Ada tombol add, delete per item
- Data disimpan sebagai array string

### 3. Buat Halaman Daftar Playbook
- Buat file `frontend/src/app/playbooks/page.tsx` untuk menampilkan daftar playbook
- Tampilkan semua playbook yang dimiliki user
- Ada tombol create new, edit, delete

### 4. Integrasi dengan Halaman Input Trade (Opsional)
- Saat user akan membuat trade baru, tampilkan checklist dari playbook yang dipilih
- Sebagai langkah konfirmasi sebelum submit trade
- Bisa dalam bentuk modal atau bagian dari form trade

### 5. Buat Service dan API Calls
- Pastikan playbook.service.ts sudah mencakup semua kebutuhan CRUD
- Tambahkan validasi jika diperlukan

### 6. Testing
- Buat test sederhana untuk memastikan fitur berjalan
- Cek API response
- Cek UI rendering

## Estimasi Waktu
- Halaman form: 2 jam
- Entry checklist input: 1 jam
- Halaman daftar: 1 jam
- Integrasi trade: 2 jam
- Testing: 1 jam
Total: ~7 jam

## Risiko
- Kebutuhan UI yang kompleks untuk checklist
- Integrasi dengan trade entry mungkin memerlukan perubahan di model trade

## Alternatif Solusi
Jika terlalu kompleks, checklist bisa ditampilkan sebagai bagian dari dashboard sebagai pengingat statis.

## Pertanyaan ke User
1. Apakah checklist ini akan ditampilkan saat akan melakukan entry trade, atau hanya sebagai perencanaan di playbook?
2. Apakah ada item checklist baku yang harus ada, atau benar-benar bebas?
3. Di halaman mana checklist ini sebaiknya ditampilkan?
